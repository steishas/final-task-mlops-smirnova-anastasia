# ADR-002: Внедрение улучшенной системы для снижения времени отклика

Дата: 2026-05-29

## Статус
Принято

## Контекст
Текущий ML-сервис демонстрирует среднее время отклика около 3.5 секунд, что значительно превышает установленный SLO (p95 < 200 мс). Предложена улучшенная архитектура инференса, которая, по предварительным оценкам, обеспечивает среднее время отклика около 2.0 секунд. Для принятия решения о внедрении используется подход Metrics Driven Development (MDD) со статистическим обоснованием.

Расчеты приведены в [EDA](model-experiments/01_eda.ipynb)

## Данные
Предоставлены два синтетических набора данных по 500 000 измерений времени отклика каждый:
- **Существующая система**: нормальное распределение с $\mu$ = 3.5 с и $\sigma$ = 0.4 c;
- **Улучшенная система**: нормальное распределение с $\mu$ = 2.0 с и $\sigma$ = 0.4 c.

Визуализация распределений подтверждает существенный сдвиг в сторону уменьшения времени отклика у улучшенной системы.

## Гипотезы
- **$H_0$** (нулевая гипотеза): Среднее время отклика улучшенной системы равно среднему времени отклика существующей системы ($\mu$_improved = $\mu$_existing)
- **$H_1$** (альтернативная гипотеза): Среднее время отклика улучшенной системы меньше, чем у существующей ($\mu$_improved < $\mu$_existing)

## Метод
Применён односторонний двухвыборочный t-тест (стьюдента) для независимых выборок. Выбор теста обоснован нормальным распределением двух выборок и равенством дисперсий. Уровень значимости $\alpha$ = 0.05. Доверительная вероятность (1 - $\alpha$) = 0.95.

## Результаты

| Метрика | Значение |
|---------|----------|
| Разница средних | -1.5003 с |
| 95% ДИ для разницы средних | [-1.5019, -1.4988] с |
| t-статистика | -1875.0793 |
| p-value | < 0.0001 |

## Решение
Полученное значение p-value < 0.0001 значительно меньше установленного уровня значимости $\alpha$ = 0.05. 95% доверительный интервал для разницы средних [-1.5019, -1.4988] секунд не включает ноль, что дополнительно подтверждает статистическую значимость улучшения.

**Нулевая гипотеза отвергается. Принимается альтернативная гипотеза.**

**Архитектурное решение: внедрить улучшенную систему**, так как она обеспечивает статистически значимое снижение времени отклика в среднем на 1.5 секунды.

## Последствия

### Положительные
- Существенное снижение времени отклика системы улучшит пользовательский опыт;
- Ожидается достижение SLO по latency (p95 < 200 мс);
- Повышение пропускной способности сервиса.

### Отрицательные
- Требуется рефакторинг или замена текущего инференс-сервера;
- Необходимо провести нагрузочное тестирование перед внедрением в продакшен;
- Возможные дополнительные затраты на вычислительные ресурсы.

### Риски
- Результаты получены на синтетических данных; требуется валидация на реальной нагрузке;
- Улучшение времени отклика не должно приводить к деградации качества предсказаний.
